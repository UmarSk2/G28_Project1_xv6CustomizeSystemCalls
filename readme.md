# xv6 System Call Extensions Project

This project modifies the **xv6-riscv** operating system to include five new system call functionalities. These modifications demonstrate an understanding of kernel-space programming, process management, and inter-process communication.

---

## 1. Directory Structure of Modified xv6
The following files are the primary locations for modifications:

```text
xv6-riscv/
├── kernel/
│   ├── syscall.h     # Defined new System Call numbers (SYS_getprocs, etc.)
│   ├── syscall.c     # Added kernel-side function pointers and names
│   ├── sysproc.c     # Implemented the logic for process/system syscalls
│   ├── proc.c        # Modified process structures (priority/states)
│   └── proc.h        # Updated struct proc to include priority or signal masks
├── user/
│   ├── user.h        # Added user-level prototypes for the new calls
│   ├── usys.pl       # Added entry points for assembly stub generation
│   └── testbench.c   # New user-level program to demonstrate functionalities
└── Makefile          # Added testbench to UPROGS for compilation
```

## 2. Implemented System Calls

### I. `getprocs` (Process Management)
* **Purpose:** Returns the total number of active processes in the system.
* **Logic:** Iterates through the global `proc` table and counts entries where `state != UNUSED`.

### II. `setpriority` (Scheduling)
* **Purpose:** Allows a process to change its own scheduling priority.
* **Logic:** Updates a `priority` field in the `proc` structure, which can be utilized by a priority-based scheduler.

### III. `sigsend` (Signals/IPC)
* **Purpose:** Sends a basic signal (integer) to a specific PID.
* **Logic:** Finds the target process in the kernel process table and sets a signal flag or handles the logic for notification.

### IV. `uptime_ms` (System Info)
* **Purpose:** Returns high-resolution system uptime in milliseconds.
* **Logic:** Reads the hardware clock frequency and converts the current tick count into milliseconds.

### V. `getmem` (Memory Info)
* **Purpose:** Returns the current memory footprint (in bytes) of the calling process.
* **Logic:** Reads the `sz` (size) field from the current process's kernel structure.

## 3. How to Build and Run

### Prerequisites
* RISC-V Toolchain (gcc-riscv64-unknown-elf)
* QEMU (qemu-system-riscv64)

### Compilation
1.  Navigate to the root directory:
    ```bash
    cd xv6-riscv
    ```
2.  Clean and compile the OS:
    ```bash
    make clean
    make qemu
    ```

### Running the Demonstration
Once xv6 has booted in the QEMU terminal, run the test program:
```bash
$ testbench
```

## 4. Analysis of System Call Flow
When a user program calls a system call like `getprocs()`:

1.  **User Space:** `user.h` provides the function prototype. `usys.S` (generated by `usys.pl`) loads the unique ID `SYS_getprocs` into the `a7` register and executes the `ecall` instruction.
2.  **Trap Handling:** The RISC-V CPU traps to the kernel's `trampoline.S`, switching the processor to Supervisor Mode.
3.  **Kernel Dispatch:** The `syscall()` function in `kernel/syscall.c` uses the value in `a7` to index the system call table and calls `sys_getprocs()`.
4.  **Result:** The return value is placed in the `a0` register, and control is returned to the user program.
